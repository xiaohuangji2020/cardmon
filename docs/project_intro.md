## 介绍

宝可梦 + 实时探索 +SRPG 回合制方格战斗（战棋）

CTB回合制

缝合数码宝贝三的刷卡强化机制

## 技术

1. Godot HD-2D （3D场景，2D贴图，贴图始终朝向摄像头）
2. 像素风，视口大小：**640 x 360**
3. 人物使用纸娃娃系统
    1.  在 Godot 中使用 `Sprite3D` (Billboard模式) + `Cutout Animation` (骨骼拼接)。将角色的身体部件（头、身、手、武器）拆开，在 Godot 动画播放器里做动作，这样换装备只需要替换对应的贴图（Sprite Texture），而不需要重画动作。
4. 宝可梦为逐帧像素画

## 详细说明

### 像素大小

- **视口大小：推荐分辨率：640 x 360**
- **像素密度 (PPU)：**
    - 设置 **1.0 米 = 32 像素**。
- **人物高度：** 48px。
- **宝可梦 (根据体型分级)：**
    - **小型 (S)：** 24 - 32 像素 (如皮卡丘)。
    - **中型 (M)：** 48 - 64 像素 (如喷火龙)。
    - **巨型 (L)：** 96 - 128 像素 (如大岩蛇)。
- **3D 格子：** 每格 1.0m (对应 32px 贴图)。
- **视角：** 3D 摄像机建议使用 **正交投影 (Orthogonal)**，FOV 设置小一点，模拟 2D 的平展感。

### 实时探索

1. 实时探索与战斗切换类似于游戏《废土3 (Wasteland 3)》，探索为正常行走，攻击后/主动展开网格进入战斗
2. 可以与npc对话
3. 遭遇战：明雷：地图上就是会有野生的宝可梦，有警戒范围（玩家离得近了可能会逃跑，可能会战斗）

### 宝可梦

1. 需要自己设计形象
2. 属性：水、火、木、冰、雷、风、土、钢、光、暗、无属性，每只宝可梦可能有1-2个属性
3. 技能：可以携带4个
4. 特性：漂浮、雨托盘、轻快、光合作用等等，每只宝可梦只有一个特性
5. 移动类型：步行、飞行、漂浮、游泳、遁地
6. 能力值：HP、攻击、防御、特攻、特防、速度，六个属性。速度决定行动力回复速度，行动力满即可行动
7. 等级：宝可梦可以升级，升级会获得能力值点数（有固定的种族值成长，少量自由分配），也可能会学会新技能
8. 可以进化：不同的进化方向（类似转职）常规进化为永久。战斗中有mega进化，为临时的
9. 闪光宝可梦

### 玩家

1. 玩家也是要上战场的，可以刷卡辅助，也可以战斗
2. 刷卡需要再目标宝可梦附近，可以通过技能树扩大这个距离
3. 玩家有自己的天赋/技能树
4. 玩家可以提取未上场的宝可梦的数据给自己强化，比如特性、主动技能（玩家装备后会不一样），仅机制，不附加宝可梦的数值
5. 玩家可以与宝可梦合体进化，类似mega进化
6. 玩家死亡则战斗失败。但要注意为玩家提供一些防御手段。避免太脆直接被灭了
7. 玩家主动攻击后获取大量同步率点数，鼓励玩家亲自战斗
8. 玩家比宝可梦的优势，可以随时提取未上场宝可梦属性、特性、技能。但伤害一般会低于宝可梦亲自战斗
9. 被提取能力的宝可梦不可上场
10. 每多召唤一只宝可梦上场，玩家能力就会下降一次。可以通过天赋/技能树改善这点

### 区别宝可梦和玩家

| **维度** | **🦁 宝可梦 (The Power)** | **🧑‍💻 训练师 (The Tactician)** |
| --- | --- | --- |
| **主要职责** | **主炮手 / 坦克** | **控制 / 充能 / 补刀 / 改变地形** |
| **伤害量级** | 高 (100% - 300%) | 低中 (40% - 60%) |
| **伤害范围** | 经常有 AOE (扇形、大圆) | 通常是单体 (点杀) 或 线性 (射击) |
| **属性依赖** | 强依赖 11 种属性克制 | **无属性 / 可以提取宝可梦的属性** |
| **生存方式** | 靠血量和防御硬抗 | 靠闪避、护盾、掩体 |
| 特点 |  | 可以随时提取未上场宝可梦能力（属性、特性、技能，但不能提取能力值） |

**举例：提取一只【火系喷火龙】的数据**

- **如果喷火龙自己上场：**
    - 技能：**喷射火焰**
    - 效果：对前方扇形区域造成 200 点火系伤害，并留下燃烧地形。
    - *特点：简单粗暴的毁灭力。*
- **如果训练师提取了数据：**
    - 技能：**火焰弹 (Fire Shot)**
    - 效果：对单体造成 50 点伤害，但**必定造成“烧伤”状态**（每回合扣血），或者**融化目标的护甲**（降低物防）。
    - *特点：功能性。训练师利用火的数据，不是为了烧死敌人，而是为了削弱敌人，方便宝可梦收割。*

即对所有可提取能力的宝可梦，单独设计一份被提取后的技能

### 同步率点数

1. 用于给下面的指令卡系统刷卡用
2. 回复方法：随着时间自然回复（少量）；宝可梦攻击/击败敌人（中等）；玩家亲自攻击（中等到大量）
3. 同步率点数存在上限
4. 同步率也用来支付合体进化。合体进化每回合消耗大量同步率。同步率归零强制解体
5. 召唤新的宝可梦也需要刷卡，也需要消耗同步率
6. 召唤的宝可梦越多，同步率上限越低且回复越慢（各种方式都变慢），同时玩家能力也会有所下降

### 指令卡系统

1. 玩家可以通过刷秘籍卡，给宝可梦学习技能
2. 玩家可以刷空白的捕捉卡，来捕捉宝可梦，捕捉到的宝可梦在当场战斗可以立即上场，但保留当前血量
3. 战斗中玩家可以像《数码宝贝三》那样刷卡，给宝可梦加上临时强化
4. 战斗中玩家可以携带自己的全部指令卡，且每张卡片使用后有冷却/充能时间，指令卡包按照功能、效果分类方便玩家查找。且玩家可以标记常用指令卡，方便自己直接找到。
5. 卡牌为永久的
6. 指令卡可以通过锻造提升（一套给指令卡的提升系统）

### 指令卡锻造系统

1. 战斗外的系统，可以随时洗点
2. （一个有趣的思路，类似购买能量贴纸贴在卡片上）
3. 基础强化 —— 纯数值提升
4. 机制重构 —— 改变功能
• 范围模组： 射程 +1。
• 形状模组： 单体攻击 → 十字范围攻击（同时伤害倍率降低至 80% 以平衡）。
• 控制模组： 攻击附加 20% 概率“麻痹”。
• 地形模组： 攻击后在地面留下“燃烧”区域。
• 吸血模组： 将造成伤害的 20% 转化为宝可梦的 HP。
5. 控制平衡 —— 容量限制 (Capacity System)
    - 每张卡都有一个 容量上限（比如 10点）。
    - 基础数值强化： 占用 1 点容量。
    - 范围+1 模组： 占用 5 点容量（非常昂贵）。
    - 附加异常状态： 占用 3 点容量。

### 捕捉系统

1. 大地图： 宝可梦可见（明雷），像《朱/紫》那样
2. 战斗中：
    - 普通捕捉： 击败敌人获得 20% 数据碎片。集齐 100% 合成。
    - 完美捕捉： 训练师如果在战斗中走到残血敌人附近，刷一张 “强制封印卡”，可以直接当场获得这只怪（省去刷碎片的麻烦），或者获得 300% 的数据碎片。
    - 有主人的宝可梦不可直接捉

### 行动力系统

1. 半回合制、CTB、跑条
2. 宝可梦根据自身的速度回复行动力
3. 移动和使用技能都会消耗行动力（移动消耗的体力呈阶梯增长）
4. 行动力不为0，就可以继续行动
5. 使用技能前只要还有行动力，就可以使用，使用后行动力可以为负数
6. 行动力小于0，会陷入*筋疲力竭*状态，行动力回复变慢，且易伤

### 地形系统

1. 燃烧地面、迷雾、草地、水域、山脉、冰面等
2. 地形可以相互改变，比如草地被火系技能打中会变成燃烧地面；水域被冰系技能打中，会变成冰面；等等

### 天气系统

1. 大晴天、雨天、沙暴、冰雹
2. 技能可以改变天气，特性也可以改变天气，不同的图也可能有不同天气
3. 不同天气有不同效果，比如让对应技能威力增加，降低命中率，每回合扣血

### 异常状态

中毒、烧伤、麻痹、迟缓、冰冻、睡眠、失明

### 胜负条件

1. 类似火焰纹章，胜利条件可以是，敌全灭、主帅击破、占领等等
2. 失败条件：己方全灭、玩家死亡

### 其他系统

1. 体型系统：宝可梦可以搭载其他宝可梦，但有体型限制（先不做，待考虑）
2. 搭载系统：同上。宝可梦可以搭载其他宝可梦，但有体型限制（先不做，待考虑）

## 核心循环

1. **探索阶段 (Real-Time Exploration):**
    - **操作：** 玩家直接控制训练师（WASD或鼠标点击），宝可梦跟随在身后。
    - **视角：** HD-2D 场景，无网格显示，画面精美，光影自然。
    - **交互：** 你可以走过去开箱子、和 NPC 对话、利用训练师的技能解谜（比如黑入电脑）。
2. **战斗触发 (Transition):**
    - 当你走进敌人的视野圈，或者触发剧情。
    - **无缝切换：** 地面立刻浮现出 **全息网格 (Holographic Grid)**。
    - 角色根据当前站位，自动“吸附”到最近的格子里。战斗 UI 弹出。
3. **战斗阶段 (Turn-Based Grid Combat):**
    - 进入标准的 CTB 回合制。
    - 移动受“行动力”限制，必须走格子。
    - 技能有明确的范围判定。

### 战斗循环说明

1. 战棋游戏，初始上场仅有玩家和一只宝可梦，初始提供部分同步率
2. 玩家可以刷卡召唤下一只宝可梦，也可以选择使用同步率做别的
3. CTB跑条的战棋游戏
4. 我放召唤宝可梦越多，同步率回复越慢，玩家越弱。
5. 玩家死亡战斗结束，玩家可以更换场上宝可梦，被收回的宝可梦在战斗结束前保留状态和血量

### 简单的战斗流程例子

1. 玩家与宝可梦同时在场，初始只有一人一兽，提供初始同步率，足够召唤第二只
2. 按照速度回复行动力，先满行动力的单位先行动
3. 移动/使用技能都会消耗行动力，只要行动力不为0，就可以继续操作
4. 只要使用技能前，行动力不为0，就可以使用出技能。技能使用后，行动力可能为负
5. 行动力为负，单位会进入“筋疲力竭”状态，行动力回复变慢
6. 玩家也可以战斗，有自己的技能树，可以使用不同的技能
7. 玩家拥有类似”暴龙机“的道具，可以给宝可梦刷强化、道具，刷卡需要在宝可梦附近才可以生效
8. 刷道具和强化需要消耗同步率点数，这个同步率点数会根据时间片自然回复，可以通过各单位攻击/击败敌人获得，玩家亲自战斗能获得大量同步率点数
9. 玩家可能会被攻击，玩家死亡则战斗失败
10. 玩家可以和宝可梦进行合体进化，类似mega进化
